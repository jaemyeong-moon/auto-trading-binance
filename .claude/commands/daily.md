일일 서버 데이터 분석 및 알고리즘 최적화를 실행한다.

## 실행 절차

### 1단계: 서버 분석 스크립트 실행
SSH로 서버에 접속하여 `scripts/daily_analysis.py`를 실행하고 JSON 리포트를 받는다.

```
ssh -i ~/.ssh/ssh-key-2026-03-25.key ubuntu@168.107.18.151 'cd ~/auto-trading-binance && source .venv/bin/activate && PYTHONPATH=. python3 scripts/daily_analysis.py'
```

### 2단계: 리포트 분석 (퀀트 관점)
JSON 리포트를 기반으로 다음을 분석한다:

- **서버 상태**: 봇/대시보드 프로세스 살아있는지, RAM/디스크 여유
- **실거래 성과**: 24시간/7일/전체 승률·PF·순수익 추이
- **페이퍼 전략 랭킹**: 14개 전략 중 eligible 전략 (5건+, WR 45%+, PF 1.0+)
- **시그널 품질**: confidence 분포, 진입 시그널 빈도
- **열린 포지션**: 실거래/페이퍼 미청산 건, 현재 손익

### 3단계: 자동 판단 및 조치
리포트의 `recommendation` 필드를 기반으로:

1. **추천 전략이 현재 전략과 다르면** → 데이터가 충분하고 우위가 명확할 때만 전환
2. **모든 전략이 부적격이면** → 현재 전략 유지, 페이퍼 데이터 축적 대기
3. **서버 문제 감지 시** → 프로세스 재시작, RAM 이슈 알림
4. **열린 실거래 포지션** → 현재가 대비 PnL 확인, SL/TP 적정성 검토

### 4단계: 알고리즘 업데이트 검토
- 승률 40% 미만 전략: 파라미터 조정 또는 비활성화 고려
- 신규 전략(v14/v15): 페이퍼 데이터 충분히 쌓였으면 성과 평가
- v13(orderflow): 데이터 주입 후 성과 개선 추이 확인

### 5단계: 결과 보고
사용자에게 한국어로 간결하게 보고:
- 주요 지표 변화 (전일 대비)
- 취한 조치 (전략 변경, 파라미터 조정 등)
- 다음 확인 시점

## 판단 원칙
- 검증 안 된 전략으로 실거래하지 않는다
- 페이퍼 5건 미만은 통계적 무의미 — 판단 보류
- 실거래 전환은 페이퍼 WR 50%+ AND PF 1.2+ AND 거래 10건+
- 현재 수익 중인 포지션은 건드리지 않는다
