## 한줄 요약

EMA 크로스로 방향을 잡고, 항상 포지션을 유지하며, 방향이 바뀌면 즉시 플립한다.
연패가 쌓이면 역추세 모드로 자동 전환한다.

---

## 전략 구조

### 진입 신호

```
EMA(3)이 EMA(8)을 상향 돌파 → LONG
EMA(3)이 EMA(8)을 하향 돌파 → SHORT
```

크로스가 발생하지 않더라도, 기존 추세 방향 + 거래량 급증이 동시에 발생하면 진입 신호로 판단한다.

| 조건 | 방향 | 신뢰도 |
|------|------|--------|
| EMA 크로스 + 거래량 급증 | LONG/SHORT | 0.9 |
| EMA 크로스 + 거래량 보통 | LONG/SHORT | 0.7 |
| 추세 유지 + 거래량 급증 | LONG/SHORT | 0.6 |
| 그 외 | HOLD | 0.0 |

### 거래량 필터

```
현재 거래량 > 최근 20봉 평균 × 1.2 → 거래량 급증(vol_strong)
```

거래량이 동반되지 않은 크로스는 낮은 신뢰도로 처리한다.

### 포지션 관리 원칙

1. **항상 포지션 보유** — 롱 아니면 숏, 빈 손 없음
2. **플립 방식** — 방향 전환 시 기존 청산 → 즉시 반대 진입
3. **같은 방향 신호는 무시** — 이미 LONG인데 LONG 신호 → HOLD

### 익절 / 손절

| 항목 | 기본값 | 대시보드에서 변경 가능 |
|------|--------|----------------------|
| 익절 (TP) | 1.0% | O |
| 손절 (SL) | 0.5% | O |

TP/SL 도달 시:
1. 현재 포지션 청산
2. 즉시 전략 재평가
3. 신호가 있으면 바로 재진입

### 역추세 모드 (Contrarian Flip)

```
3연패 → 역추세 모드 진입 (신호 반전)
역추세 모드에서 1회 수익 → 정상 모드 복귀
```

**왜 필요한가:**
단방향 추세에서 반복적으로 반대 방향을 잡으면 연패가 발생한다.
이때 신호를 뒤집으면, 오히려 추세에 올라타는 효과가 있다.

**동작 예시:**
1. SHORT → 손절, SHORT → 손절, SHORT → 손절 (3연패)
2. 역추세 모드 진입
3. 다음 신호가 SHORT → 반전하여 LONG 진입
4. LONG 수익 → 정상 모드 복귀

## 엔진 동작 흐름

```
매 틱 (기본 30초)
  │
  ├─ 1분봉 100개 조회
  │
  ├─ 열린 포지션 있으면
  │   ├─ TP 도달? → 청산 → 재진입 판단
  │   ├─ SL 도달? → 청산 → 재진입 판단
  │   └─ 아니면 계속
  │
  ├─ 전략 평가 (MomentumFlipScalper.evaluate)
  │   ├─ HOLD → 대기
  │   └─ BUY/SELL → 기존 청산 + 반대 진입 (플립)
  │
  └─ 손익 기록 → ScalperState 업데이트
```

## 파라미터 (대시보드 설정)

| 파라미터 | 키 | 기본값 | 설명 |
|----------|----|--------|------|
| 투자 비율 | `position_size_pct` | 0.1 (10%) | 잔고 대비 1회 투입 비율 |
| 레버리지 | `leverage` | 5 | 선물 레버리지 배수 |
| 익절 | `tp_pct` | 0.01 (1%) | 가격 변동 기준 |
| 손절 | `sl_pct` | 0.005 (0.5%) | 가격 변동 기준 |
| 분석 주기 | `tick_interval` | 30초 | 시장 분석 빈도 |
| EMA Fast | 코드 내 고정 | 3 | 빠른 이동평균 기간 |
| EMA Slow | 코드 내 고정 | 8 | 느린 이동평균 기간 |
| 거래량 임계 | 코드 내 고정 | ×1.2 | 평균 대비 거래량 배수 |
| 역추세 발동 | 코드 내 고정 | 3연패 | 연속 손실 횟수 |

## 알려진 한계

1. **횡보장에서 약함** — EMA 크로스가 빈번하게 발생하여 잦은 플립 → 수수료 손실 누적
2. **거래량 필터가 단순함** — 20봉 평균 × 1.2는 시장 상황에 따라 너무 느슨하거나 타이트할 수 있음
3. **역추세 모드 진입 기준이 고정** — 3연패가 최적인지는 검증 필요
4. **EMA 기간이 고정** — 변동성에 따라 동적으로 조절하면 성능 개선 가능성 있음
5. **수수료 미반영** — 전략 평가 시 Binance 수수료(0.02~0.04%)를 고려하지 않음
