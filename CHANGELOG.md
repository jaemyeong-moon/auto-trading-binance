# Changelog

## [0.7.0] - 2026-04-05

### テーマ: Phase 7 전략 고도화 — 데이터 기반 매매 전략 최적화

**실거래 + 가상매매 데이터를 분석하여 전략별 약점을 정량화하고, 시간대 필터·SL/TP 비율·진입 조건을 데이터 기반으로 개선했습니다.**

---

#### 1. DB 기반 동적 시간대 필터

**今まで**: v11(DataDrivenScalper)만 `NO_TRADE_HOURS_KST = set(range(0, 10))`으로 하드코딩된 시간대 필터를 갖고 있었습니다. 나머지 6개 전략은 새벽 시간대(승률 10%)에도 무방비로 거래했습니다.

**今後**: `src/core/time_filter.py`로 전 전략이 공유하는 DB 기반 동적 필터를 도입했습니다. 실거래+가상매매 기록에서 시간대별 승률을 자동 계산하고, 승률 35% 미만 & 거래 3건+ 시간대를 차단합니다. 캐시 TTL 1시간으로 DB 부하를 최소화하며, DB가 비어있으면 기존 0~9시 차단으로 폴백합니다.

```python
# 모든 전략의 evaluate() 최상단에 자동 적용
from src.core.time_filter import is_tradeable_hour
if not is_tradeable_hour():
    return self._hold(symbol, reason="blocked_hour")
```

#### 2. SL/TP 배수 전략별 최적화

**今まで**: v1은 SL:TP = 1:1 (2.0:2.0 ATR)로 SL 체결이 TP의 1.8배였고, v3는 SL 1.0 ATR로 노이즈에 의한 조기 손절이 빈번했습니다.

**今後**: 전 전략에 최소 RR 1:1.5를 보장하도록 SL/TP 배수를 재조정했습니다.

| 전략 | SL (Before→After) | TP (Before→After) | RR |
|------|-------------------|-------------------|-----|
| v1 | 2.0 → 2.5 | 2.0 → 3.75 | 1:1.5 |
| v3 | 1.0 → 1.5 | 2.0 → 3.0 | 1:2 |
| v10 | 1.5 → 2.0 | 2.25 → 3.0 | 1:1.5 |
| v12 | 3.0 → 2.5 | 5.0 (유지) | 1:2 |

auto_optimizer의 SL 탐색 하한도 5.0→6.0으로 상향하여 너무 타이트한 SL 조합을 제거했습니다.

#### 3. 진입 신호 강화 — LONG 기준 상향

**今まで**: Phase 6 분석에서 LONG 평균 손실이 SHORT 대비 5배 컸지만(avg -1.83 vs -0.34), LONG/SHORT에 동일한 진입 기준이 적용되고 있었습니다.

**今後**: 전 전략에 LONG 진입을 더 까다롭게 만드는 비대칭 조건을 적용했습니다.

- **v1**: LONG 진입 시 거래량 1.2배 이상 필수 (미충족 시 HOLD)
- **v3**: LONG score threshold 5→6 (7점 만점)
- **v9**: LONG RSI 상한 60→55 (과매수 진입 방지)
- **v10**: LONG 과매도 기준 RSI < 35 → < 30 (더 극단적 바닥에서만 진입)
- **v12**: LONG 패턴 confidence에 10% 페널티 적용

#### 4. 거래별 SL/TP 프로그레스바

**今まで**: 거래 내역은 테이블 형태로만 표시되어, 각 거래가 SL/TP 중 어디 근처에서 청산됐는지 직관적으로 파악하기 어려웠습니다.

**今後**: 거래 내역 페이지와 시뮬레이션 탭에 프로그레스바 형태의 시각화를 추가했습니다. API 호출 없이 DB 데이터만으로 즉시 렌더링됩니다.

```
SL $42,900 ████████████▼진입██████████●청산██████ TP $43,300
           ◄── 빨강(손실) ──►◄──── 초록(수익) ────►
```
